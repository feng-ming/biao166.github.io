1月 10 2011

.NET搭建IIS提示：Logon failure: user account restriction.

Logon failure: user account restriction. Possible reasons are blank passwords not allowed, logon hour restrictions, or a policy restriction has been enforced

解决方案:针对windows xp

1、运行gpedit.msc到组策略管理界面下,计算机配置—->Winsows设置——->安全设置—->本地策略 —->用户权利指派,看看右边有一行:“拒绝从网络访问这台计算机 ”看它的属性里有没有guest一项,若有,则删除.

2、 若还不行,在我的电脑窗口里工具—->文件夹选项——>查看——->高级选项里有“使用简单文件共享” 打勾去掉,确定下去,.然后再访问.

3、开始—运行—输入gpedit.msc回车—计算机配置—windows设置—安全设置—本地策略—安全设置—“账户：使用空白密码的本地账户只允许进行控制台登录”—改为“已禁用”。

ps：留着以后用，最怕搭建环境是出现错误。

Microsoft JET Database Engine 错误80004005 未指定的错误的解决

重装了系统，IIS却出现了80004005错误》》》

这个错误算是比较常见了，

  • 但由于之前源码测试正常，所以排除数据库连接错误；
  • access绝对的正常，之前全是用这个，所以排除；
  • 最后剩下一个目录权限：
    目录权限，把程序所在文件夹设为everyone后还是没反应，但网络有人说可能是temp文件夹搞的鬼，试了一下，把temp的现在所登录的用户添加修改权限，额，竟然好呢，得来全不费功夫。
