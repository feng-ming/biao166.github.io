CSS布局自适应高度解决方法

在修改webbyps.info的主题时，遇到了之前没有想到的问题，而问题的根源就是左右两栏的高度自适应问题，纠结。。。。但最终还是找到了解决的方法。比较无语的是，之前我知道，但就是没想到。

代码需求环境：典型的三行二列布局，每列高度（事先并不能确定哪列的高度）的相同，是每个设计师追求的目标，按一般的做法，大多采用背景图填充、加JS脚本 的方法使列的高度相同，本文要介绍的是采用容器溢出部分隐藏和列的负底边界和正的内补丁相结合的方法来解决列高度相同的问题。

先看代码：



wrap{


 overflow: hidden;
 }


sideleft, #sideright{


 padding-bottom: 32767px;
 margin-bottom: -32767px;
 }

实现原理：

块元素必须包含在一个容器里。

应用overflow: hidden 到容器里的元素。

应用 padding-bottom（足够大的值）到列的块元素 。

应用margin-bottom（足够大的值）到列的块元素。

padding-bottom将列拉长变的一样高，而负的margin-bottom又使其回到底部开始的位置，同时，溢出部分隐藏掉了。

利用word保存网页中大量图片（并保持原有顺序）

当我们平时浏览网页逛论坛的时候，经常会遇到不少好图片，想予以收藏。通常我们的做法是在图片上右键——另存为进行保存，或者直接按住左键拖动图片到某个文件夹，这对于保存少量图片的确是很方便，如果遇到了大量的图片呢，会存图存到手抽筋的。

对于多图，比较传统的方法就是把网页直接保存，然后再到保存的文件夹里找自己想要的图，或者用迅雷下载全部链接，文件格式选图片的。但这些方法都还得自己在一堆图片中寻找自己所想要的，图片不按顺序排列且文件命名混乱。

我想我追求的是一种一次性批量保存并且能保持原有顺序的高速快捷方法。我这里讲的方法是用Word2007（word2003也没关系，不过要装一个2003-2007的升级补丁）。

为什么2007可以，而2003不可以呢

MicrosoftOfficeWord2007提供了一种默认文件格式，叫做MicrosoftOfficeWordXML格式（docx的x就是XML）。这种新的文件格式将文件分成一些文档部件，其中每个部件定义了文件的部分内容，当您希望更改文件当中的一些内容的时候，您可以简单的查找您希望更改的文档部件，图片是其中的一种部件，利用这个原理我们来保存图片。也就是我们利用winrar软件读取出这个里边的文件夹，再保存出来，看着有点复杂，但操作起来还是挺快捷的。

免费图片工具Picasa 十个使用技巧挖掘

Picasa来自Google的免费图片管理工具，Picasa3这个新版本里不断加入了很多有用的功能，能提高用户的图片管理效率。很多人都用不惯这个漂洋过海来的软件，因为可能与我们国人的理解模式不一样，但它却是我用过第二强大的图片管理软件（第一强还没发现）。
我们在用PICASA的时候，经常遇到一些问题，解决了下面的问题后，你应该会用的顺手多了

1、问题：图片哪里去了?

这可能是由3个原因造成的：

(1)可能图片大小小于250*250像素。Picasa默认不显示小于这个尺寸的图片，所以有时你会发现无论如何都找不到这些图片。要让这些图片显示出现只需点击【视图】，勾选“显示小图片”。

(2)可能是Picasa不支持图片的格式，或你没有勾选Picasa显示某种格式的图片。你可以在【工具】-【选项】-【文件类型】里选择显示的格式。

(3)可能是你的图片不在Picasa扫描的范围。你可以在【工具】-【文件夹管理器】里进行设置picasa要扫描的文件夹。

2、批量重命名

选中所有要重命名的图片，按下F2即可。（好像windows里边都一样吧）

3、Picasa不会搜索隐藏文件(夹)

所有包含隐藏属性的文件或文件夹Picasa都不会扫描的。所以，你不用害怕你的绝密照片被Picasa暴露。
